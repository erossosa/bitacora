SEC‑01 · Esqueleto mínimo de menú con case

Referencia
$ read -p "Opción: " op
$ case "$op" in
  1)  # acción A ;;
  2)  # acción B ;;
  3)  # salir    ;;
  *)  echo "Opción inválida" ;;
$ esac
  • Patrones: 1, 2, 3, *.
  • Cada rama cierra con ;;. El bloque cierra con esac.
  • Pedí datos dentro de cada rama con read -p.

SEC‑02 · Operaciones típicas de administración

Grupos
$ sudo groupadd nombre_grupo      # crear grupo
$ getent group nombre_grupo       # ver info del grupo
Usuarios
$ sudo useradd -m nombre_usuario   # crear usuario con home
$ id nombre_usuario               # verificar usuario
$ sudo usermod -aG nombre_grupo nombre_usuario  # agregar a grupo
Naming: minúsculas, sin espacios. Ej.: devs, qa, juanp.

SEC‑03 · Validación y verificación

Validación
$ id juanp
$ getent group devs
$ cat /etc/passwd    # usuarios (uno por línea)
$ cat /etc/group     # grupos y miembros
$ grep '^juanp:' /etc/passwd   # filtro rápido
$ grep '^devs:'  /etc/group

Mensajes claros tras cada acción: “✓ creado”, “✗ ya existe”, “entrada inválida”.

SEC‑04 · Flujo sugerido (pseudocódigo)

Diseño
leer opción
case opción
  1) pedir nombre de grupo -> si vacío: avisar -> crear -> informar -> (opcional) getent
  2) pedir nombre de usuario -> si vacío: avisar -> crear -m -> informar -> (opcional) id
  3) salir con mensaje amable
  *) inválida -> ayuda breve y volver al menú
esac

SEC‑05 · Errores comunes

  • Olvidar ;; al final de una rama → syntax error.
  • No cerrar con esac.
  • Crear algo que ya existe → informar y continuar (no te caigas).
  • Mayúsculas/espacios en nombres → evitá problemas futuros.

SEC‑06 · Checklist exprés

  • [ ] Menú con todas las opciones.
  • [ ] Pide lo mínimo necesario y confirma.
  • [ ] Maneja opción inválida con *.
  • [ ] Validación con id / getent / cat.

SEC‑07 · Glosario mini

  • case: selector por patrones (cierra con esac).
  • groupadd: crea grupos del sistema.
  • useradd -m: crea usuario con home.
  • usermod -aG: suma a grupo(s) sin reemplazar.
  • id: info de usuario (UID/GID, grupos).
  • getent group: consulta grupos y miembros.
  • /etc/passwd y /etc/group: bases de usuarios/grupos.
